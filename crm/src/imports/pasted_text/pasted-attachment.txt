markdown# WORK CRM — Полная спецификация фронтенда

## Контекст
Персональная CRM для отслеживания поиска работы. Один пользователь.
Данные из Supabase (прямые запросы, без бэкенда).
Стиль: между Linear и Google — чистый, минималистичный, функциональный.
Референсы: Linear, Notion, Google Workspace.

## Стек
- React 18 + React Router v6
- Supabase JS client (@supabase/supabase-js)
- Tailwind CSS
- React Query (@tanstack/react-query) — кэш и рефетч
- Supabase Auth — email/password, один пользователь
- Recharts — графики в аналитике
- @dnd-kit — drag & drop в настройках воронки

## Темы
Светлая и тёмная с переключателем. Хранится в localStorage.
ThemeContext оборачивает всё приложение.

## Адаптивность
- Desktop (1024px+): полный функционал
- Tablet (768px): адаптированная таблица
- Mobile (375px): карточки вместо таблицы, bottom sheet меню

## НАВИГАЦИЯ

Одна строка вверху на всех страницах:
WORK · [Вакансии] [Аналитика] [Настройки] · [hh↗] [tg↗] [сайт↗] [🔍] [🌙]

- WORK — логотип, клик → /dashboard
- Вакансии / Аналитика / Настройки — активный таб подчёркнут

- tg↗ — открывает Telegram
- сайт↗ — открывает andrey-mikhaylichenko.ru
- 🔍 — открывает поиск (input появляется inline, ищет по компании и роли)
- 🌙 — переключатель темы

На мобиле: WORK слева, гамбургер справа. Меню — bottom sheet снизу.

## ЭКРАН 2 — /dashboard

### Метрики (4 карточки в ряд)
[Всего вакансий] [Горячих (70+)] [Отправлено] [Получено ответов]
Каждая: большая цифра + подпись + маленький тренд ↑↓ по сравнению с прошлой неделей.

### Воронка (горизонтальная, под метриками)
Этапы берутся из таблицы `pipeline_stages` (настраиваемые).
Каждый этап: название + количество вакансий.
Клик на этап → фильтрует таблицу ниже по статусу.
Цвет каждого этапа из БД.
Между этапами — стрелка с процентом конверсии.

### Фильтры (строка под воронкой)
[Категория ▾] [Статус ▾] [Источник ▾] [Период ▾] [Поиск...] [Сбросить]
- Все фильтры сохраняются в URL params (?category=горячая&status=new)
- Сбросить — очищает все фильтры

### Таблица вакансий
Колонки (слева направо):
1. Score — цветной бейдж (🔴 70+ / 🟡 40-69 / ⚫ 0-39)
2. Компания — название
3. Роль — должность
4. Зарплата — строкой
5. Статус — dropdown прямо в ячейке, меняется без перезагрузки
6. Дата — относительная (2 дня назад)
7. → — иконка открыть

Поведение:
- Клик по строке → side panel на десктопе / переход на /vacancy/:id на мобиле
- Строки с категорией "горячая" — лёгкая красная подсветка левой границы
- Сортировка кликом на заголовок колонки
- Пагинация: 20 строк, кнопки вперёд/назад
- Skeleton loader пока загружается

На мобиле таблица → карточки:
🔴 85  Компания
Роль · Зарплата
[Статус ▾]          [→]

### Side Panel (десктоп, 480px, выезжает справа)
Поверх таблицы. URL меняется на /dashboard?vacancy=uuid.
Закрытие: крестик или Escape или клик вне панели.

Шапка панели (фиксированная):
- Компания крупно + иконка ссылки на сайт компании
- Роль
- Score бейдж + категория + зарплата + источник

Три таба: [Анализ] [Письмо] [Действия]

**Таб Анализ:**
- Score визуально — горизонтальный прогресс-бар с цветом
- Reason от агента — текст
- Два блока рядом:
  - ✅ Что совпадает (парсится из reason или отдельное поле)
  - ❌ Что не совпадает
- Кнопка "Переанализировать" — отправляет webhook в n8n

**Таб Письмо:**
- Если letter_edited есть → показываем его
- Если нет → показываем letter
- Режим просмотра по умолчанию
- Клик на текст → inline редактирование (textarea)
- Иконка копировать в правом верхнем углу текста
- Кнопка "Сохранить правки" → пишет в letter_edited
- Плашка "Оригинал" если показываем отредактированную версию + кнопка вернуть оригинал

**Таб Действия:**
- Статус — большой dropdown со всеми статусами (с цветами)
- Заметки — textarea, автосохранение через 1.5 секунды
- Следующий шаг — input текст
- Дата напоминания — datepicker
- Кнопка сохранить шаг
- Внизу: кнопка "Удалить вакансию" (серая, с модалкой подтверждения)

## ЭКРАН 4 — /analytics

### Воронка детально
Горизонтальная с конверсиями:
Добавлено(45) —80%→ Отправлено(36) —25%→ Ответ(9) —33%→ Интервью(3)

1. Вакансии по времени — линейный график (добавлено за 30 дней)
2. Score distribution — гистограмма (0-39 / 40-69 / 70-100)
3. Конверсия по источнику — таблица (HH / TG / LinkedIn — отправлено/ответы/%)
4. Топ компаний по score — горизонтальный bar chart топ-10

Переключатель периода: Неделя | Месяц | Всё время

## SUPABASE ЗАПРОСЫ

### Главный запрос (dashboard)
```sql
SELECT 
  v.*,
  c.name as company_name,
  c.site as company_site,
  c.branch as company_branch,
  va.score,
  va.category,
  va.reason,
  va.letter,
  va.letter_edited,
  va.model
FROM vacancies v
LEFT JOIN companies c ON v.company_id = c.id
LEFT JOIN vacancy_analysis va ON va.vacancy_id = v.id
ORDER BY v.created_at DESC
```

### Обновления
```javascript
// Статус
supabase.from('vacancies').update({ status }).eq('id', id)

// Заметки
supabase.from('vacancies').update({ notes, next_action, next_action_at }).eq('id', id)

// Письмо
supabase.from('vacancy_analysis').update({ letter_edited }).eq('vacancy_id', id)
```

### Метрики
```javascript
supabase.from('vacancies').select('status', { count: 'exact' })
supabase.from('vacancy_analysis').select('category', { count: 'exact' })
```

### Realtime
```javascript
supabase.channel('vacancies').on('postgres_changes', 
  { event: 'INSERT', schema: 'public', table: 'vacancies' }, 
  callback
).subscribe()
```

## UX ДЕТАЛИ

- Toast уведомления при каждом сохранении (успех/ошибка)
- Skeleton loader пока данные загружаются
- Пустое состояние с подсказкой "Добавь первую вакансию"
- Escape закрывает side panel
- Все фильтры в URL params
- Realtime — новая вакансия появляется без перезагрузки
- Автосохранение заметок через 1.5 секунды после последнего символа
- Confirm модалка перед удалением
- Относительные даты ("2 дня назад", "сегодня")
